How it Works

Three steps to deploy

Connect your GitHub account once. After that, deploy any repository through a simple conversation with Claude.

1

Install the App

Add the amonware-app GitHub App to your account. Choose which repositories to grant access to — you stay in full control.

2

Authorize via OAuth

Complete a one-time OAuth flow to link your GitHub identity. Your credentials are never stored on the client — everything runs server-side.

3

Deploy with Claude

Open Claude Desktop and say "deploy my-repo". The MCP tools handle cloning, building, and deploying — all through natural language.

Features

Built for developers

Secure, fast, and designed to work seamlessly with your existing workflow.

🔒

No PAT Required

Uses GitHub App installation tokens instead of Personal Access Tokens. No secrets to manage or rotate.

⚡

10-Second Deploys

Runtime-based deployment skips image builds entirely. Your code is mounted and running in seconds, not minutes.

🤖

MCP Native

Three dedicated tools — check_github_connection, list_user_repos, and deploy_user_repo — available right in Claude Desktop.

🗂

Granular Permissions

Select exactly which repositories the app can access. Revoke anytime from your GitHub settings.
